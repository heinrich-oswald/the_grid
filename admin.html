Help & Tutorial

A quick guide to connect and sync.

  1. Open Admin Panel at http://localhost:8000/admin.html.
  2. The site now uses a single cloud API base defined in assets/config.js (API_BASE_URL). No LAN/local host/port config is needed.
  3. If your backend requires a token, paste it in Admin Token then click Save. This adds Authorization: Bearer to all admin requests.
  4. Click Check DB Health. Expected: Online and DB: file. If not, see Troubleshooting below.
  5. Make a quick change (e.g., disable timers or set a label). Other pages should reflect changes instantly via SSE.
  6. To reset local overrides, use Reset Overrides then reload.
Tip: The Admin API broadcasts updates via server‑sent events (SSE). All devices pointing to the same API_BASE_URL receive settings instantly and store them in localStorage.admin_settings for quick reads.
Troubleshooting
  • Health fails: Ensure your Admin API is reachable at API_BASE_URL and token (if set) matches.
  • SSE error: A red error for /api/admin/stream can appear until the API connects; it clears once health is OK.
  • Reset config: Click Reset Overrides or in DevTools run localStorage.clear() and reload.
